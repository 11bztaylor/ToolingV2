Cross-Workspace Queries

Generate adx() proxy queries for use in Log Analytics / Sentinel workspaces.

What Are Cross-Workspace Queries? ▼
The adx() function lets Sentinel / Log Analytics users query data that lives in an ADX cluster without moving it. Your analysts run these queries from their LA workspace and get back results from ADX tables in real time.
How it works: LA sends the query to ADX via a cross-service proxy. The ADX cluster executes it and returns results to the LA workspace. No data replication, no data movement — just a remote query.
Prerequisite: The ADX cluster must be linked to the LA workspace. In Azure Portal: Log Analytics workspace → Settings → Linked services → add your ADX cluster. Without this, adx() calls will fail with a permissions error.
Format: adx("https://{cluster}.{region}.kusto.windows.net/{database}").{TableName}
